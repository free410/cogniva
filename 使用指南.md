# 知识问答系统 - 使用指南

## 项目简介

这是一个基于 RAG（检索增强生成）+ 间隔重复记忆系统的智能知识问答平台。

**核心功能**：
- 文档上传与智能分块
- 语义检索与 RAG 问答
- 间隔重复记忆系统
- 多模型对话（DeepSeek/OpenAI/Claude/Ollama）
- 数据统计与分析

---

## 快速开始

### 环境要求

- Python 3.10+
- Node.js 18+
- Docker + Docker Compose
- PostgreSQL 15+ (带 pgvector 扩展)

### 一键启动

```bash
# 1. 进入项目目录
cd 知识问答系统

# 2. 启动数据库
docker-compose up -d

# 3. 配置环境变量
cp .env.example backend/.env
# 编辑 backend/.env 填入 API Key

# 4. 启动后端
cd backend
pip install -r requirements.txt
uvicorn main:app --reload --port 8000

# 5. 启动前端（新终端）
cd frontend
npm install
npm run dev
```

### 访问应用

- 前端界面：http://localhost:3000
- API 文档：http://localhost:8000/docs
- 健康检查：http://localhost:8000/api/health

## 使用教程

### 第一步：上传文档

1. 点击左侧菜单 **"文档"**
2. 点击上传区域或拖拽文件
3. 支持格式：
   - PDF
   - Word (.docx, .doc)
   - Excel (.xlsx, .xls, .csv)
   - 文本文件 (.txt, .md)
4. 等待处理完成（通常几秒钟）

---

### 第二步：智能问答

1. 点击 **"问答"** 返回聊天界面
2. 在底部输入框输入问题
3. **可选配置**：
   - **RAG 开关**：启用/禁用文档检索增强
   - **模型选择**：切换不同的 LLM
   - **流式响应**：实时显示生成过程
4. 点击 **"发送"** 或按 `Enter`

**示例问题**：
```
这个文档主要讲了什么？
总结一下第三章的核心内容
对比 A 和 B 的优缺点
```

---

### 第三步：记忆管理

#### 查看统计

点击 **"记忆"** → 顶部卡片显示：
- 总记忆数
- 待复习数量
- 总复习次数
- 平均复习次数

#### 复习记忆

1. 切换到 **"待复习"** Tab
2. 点击记忆卡片
3. 评分（0-5分）：
   - 0 = 完全忘记
   - 1 = 几乎忘记
   - 2 = 记忆模糊
   - 3 = 基本记住
   - 4 = 记住
   - 5 = 完全记住

**算法原理**（SM-2 间隔重复）：
- 答对 → 下次复习间隔延长
- 答错 → 下次复习间隔缩短
- 重要性高的记忆复习更频繁

#### 创建记忆

1. 点击 **"创建记忆"** 按钮
2. 填写内容
3. 选择分类
4. 调整重要性
5. 保存

## 问题排查

### 后端启动失败

```bash
# 检查端口占用
netstat -ano | findstr :8000

# 检查数据库连接
cd backend
python -c "from core.database import engine; engine.connect()"
```

### 前端编译错误

```bash
# 清理缓存
rm -rf frontend/.next
npm install
npm run dev
```

### 文档上传失败

检查项：
1. 数据库是否运行：`docker ps`
2. 文件格式是否支持
3. 文件大小是否超限（默认 50MB）
4. 查看后端控制台错误信息

## 最佳实践

### 1. 文档管理
- 按主题分类上传文档
- 文件名清晰易懂
- 定期清理过期文档

### 2. 记忆卡片
- 一条记忆一个知识点
- 内容简洁明确（20-200 字）
- 及时复习，避免堆积

### 3. 问答技巧
- 问题具体明确
- 开启 RAG 获取文档相关回答
- 关闭 RAG 进行通用问答

### 4. 模型选择
- **DeepSeek**：中文友好，性价比高
- **Claude**：长文本，推理能力强
- **GPT-4**：综合能力强
- **Ollama**：本地部署，隐私安全

## 技术架构

### 后端技术栈
- FastAPI - 高性能 Web 框架
- SQLAlchemy - ORM 数据库操作
- PostgreSQL + pgvector - 向量数据库
- LangChain - LLM 应用框架

### 前端技术栈
- Next.js 14 - React 框架
- Zustand - 状态管理
- Tailwind CSS - 样式框架
- React Markdown - Markdown 渲染

### RAG 流程
1. 文档上传 → 文本提取
2. 文本分块（智能分割 + 重叠）
3. Embedding 生成向量
4. 向量存储到 pgvector
5. 用户查询 → 相似度检索
6. 上下文构建 → LLM 生成回答

### 间隔重复算法（SM-2）
```
基础间隔表：
第1次复习 → 1天后
第2次复习 → 3天后
第3次复习 → 7天后
第4次复习 → 14天后
第5次复习 → 30天后

根据复习质量动态调整：
- 质量 >= 3：延长间隔
- 质量 < 3：缩短间隔

根据重要性调整：
- 重要性高：复习更频繁
- 重要性低：复习间隔更长
```
